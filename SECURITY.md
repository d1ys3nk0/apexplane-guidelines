# Security Policy

## Supported versions

APG is pre-1.0. Security fixes are provided for the latest released version only.

## Reporting a vulnerability

Please do not report security vulnerabilities through public issues or discussions. Report them through a [private GitHub security advisory](https://github.com/d1ys3nk0/apexplane-guidelines/security/advisories/new), including:

- a description of the issue and its impact;
- affected versions or commits;
- steps to reproduce or a proof of concept; and
- any suggested mitigation, if available.

Maintainers will acknowledge reports promptly, investigate them, and coordinate disclosure and remediation with the reporter when appropriate.

## Scope

Reports are in scope when they affect ApexPlane Guidelines source code or published releases. Configuration, module registries, and credentials belonging only to a downstream repository should be reported to that repository’s maintainers.
